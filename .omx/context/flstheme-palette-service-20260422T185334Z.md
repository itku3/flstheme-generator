# 컨텍스트 스냅샷: FL Studio `.flstheme` 팔레트 생성 서비스

- 작업 설명: `SPEC.md`를 기반으로 FL Studio `.flstheme` 팔레트 생성기 구현 계획을 수립한다.
- 원하는 결과: MVP 구현을 시작할 수 있는 기술 구조, 자동 매핑 알고리즘, 테스트 전략, UI 흐름이 포함된 한국어 consensus plan.
- 알려진 사실/근거:
  - `SPEC.md`는 deep-interview 결과로 작성된 실행 스펙이며 최종 ambiguity는 13%다.
  - 기존 근거 문서는 `docs/context.md`, `docs/interview.md`에 있다.
  - 샘플 `Grape.flstheme` 분석상 파일은 CRLF 텍스트 `Key=Value` 구조이며 색상은 signed 24-bit RGB 정수로 보인다.
  - 주요 MVP 기능은 팔레트 선택/입력, 자동 색상 매핑, 텍스트 가독성 보정, 이미지 중심 미리보기, `.flstheme` 생성/다운로드다.
- 제약:
  - `.flstheme` 포맷은 공식 공개 표준으로 확인되지 않았다.
  - 검증된 템플릿 기반 생성이 안전하다.
  - 생성 파일은 CRLF 줄바꿈을 유지하는 편이 안전하다.
  - 계정, 로그인, 서버 저장, 공개 공유, 검색, 좋아요/댓글, 이미지 업로드, 사용자 템플릿 업로드, 모바일 최적화는 MVP 비목표다.
  - 인터뷰/계획 문서는 한국어로 작성한다.
- 미해결 질문:
  - 기본 템플릿으로 사용할 `.flstheme` 파일의 최종 선택.
  - 실제 FL Studio 로드 테스트에 사용할 FL Studio 버전.
  - 후속 공유/커뮤니티 기능의 시작 범위.
- 예상 코드베이스 접점:
  - 현재 저장소는 앱 코드가 없는 greenfield 상태다.
  - 구현 시 프론트엔드 앱, 팔레트 입력 UI, 색상 변환/매핑 도메인 로직, 미리보기 렌더링, 파일 생성/다운로드 로직, 테스트 구성이 필요하다.
